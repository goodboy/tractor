# Plan: Subinterpreter spawn backend for tractor

## Context

GH issue #379 requests subinterpreter (subint) spawning
support. Currently `tractor/spawn/_spawn.py` is a monolithic
~848-line file containing ALL spawn backends (trio subprocess,
multiprocessing) plus shared helper code. This work:

1. Modularizes `_spawn.py` by extracting backends to own submods
2. Adds a new `_subint.py` backend using Python 3.14+
   `concurrent.interpreters`
3. Wires the test harness to accept `--spawn-backend subint`

Branch: `subint_spawner_backend` (already checked out).

## Step 1: Extract `trio_proc()` to `_trio.py`

**Files:**
- `tractor/spawn/_trio.py` (new) - receives `trio_proc()`
  (lines 448-670 of `_spawn.py`)
- `tractor/spawn/_spawn.py` - remove `trio_proc()`, import
  from `._trio` at bottom of file for `_methods` dict

**Details:**
- `_trio.py` imports shared helpers from `._spawn`:
  `soft_kill`, `hard_kill`, `cancel_on_completion`
- `_trio.py` keeps `sys` import for `spawn_cmd` construction
- The `_methods` dict and its backend imports move to the
  BOTTOM of `_spawn.py` (after all function defs) to avoid
  circular import issues
- Remove `from ._entry import _mp_main` from `_spawn.py`
  (only used by mp backend)

## Step 2: Extract `mp_proc()` to `_mp.py`

**Files:**
- `tractor/spawn/_mp.py` (new) - receives `mp_proc()` +
  `proc_waiter()` (platform-specific)
- `tractor/spawn/_spawn.py` - remove `mp_proc()` +
  `proc_waiter()`, update `_methods` dict imports

**Details:**
- `_mp.py` imports `_mp_main` from `._entry` (moved from
  `_spawn.py`)
- `_mp.py` accesses `_ctx` and `_spawn_method` globals via
  `from . import _spawn` then `_spawn._ctx` (same pattern as
  `_entry.py` already uses)
- `_mp.py` gets the platform-specific `proc_waiter()` since
  it's only used by `mp_proc()`
- After this step `_spawn.py` contains ONLY:
  `SpawnMethodKey`, globals (`_ctx`, `_spawn_method`),
  `try_set_start_method()`, `new_proc()`,
  `exhaust_portal()`, `cancel_on_completion()`,
  `hard_kill()`, `soft_kill()`, and the `_methods`
  dispatch dict

## Step 3: Add `_subint.py` with `SubintProcess` + `subint_proc()`

**File:** `tractor/spawn/_subint.py` (new)

**`SubintProcess` wrapper class** - provides process-like
interface over a subinterpreter + OS thread pair:
- `.poll()` -> `int|None` (mirrors
  `subprocess.Popen.poll()`)
- `.returncode` property
- `.is_alive()` -> `bool`
- `.terminate()` -> calls `interp.close()`
- `.kill()` -> same as terminate (no OS signal for threads)
- `async .wait()` ->
  `trio.to_thread.run_sync(thread.join)`
- `.join()` -> sync `thread.join()`
- `.stdin/.stdout/.stderr` -> `None` (so `hard_kill`
  stream close is a no-op)
- Version-gated: `sys.version_info >= (3, 14)` check at
  top

**`subint_proc()` async function** - same signature as
`trio_proc()` / `mp_proc()`:
1. Create interpreter via `interpreters.create()`
2. Bootstrap via
   `threading.Thread(target=interp.exec, args=(bootstrap_code,))`
   where `bootstrap_code` is a string that imports and
   calls `_subint_main()` from `_entry.py` with the
   actor's name, uuid, parent addr, loglevel
3. Wait for peer connection:
   `ipc_server.wait_for_peer()`
4. Send `SpawnSpec` over IPC channel
5. Register in `actor_nursery._children`
6. Signal `task_status.started(portal)`
7. Wait for join, then `soft_kill` / hard-kill sequence
8. In `finally`: if thread alive, `interp.close()` +
   `thread.join(timeout)`

**Why `interp.exec(code_str)` instead of
`interp.call_in_thread()`:**
- `call_in_thread()` requires the callable to be
  shareable across interpreters which has restrictions
- `interp.exec()` with a code string that does
  `from tractor.spawn._entry import _subint_main; _subint_main(...)`
  is more reliable since the subint re-imports fresh

## Step 4: Wire up new backend + add entry point

**Files:**
- `tractor/spawn/_spawn.py`:
  - Extend `SpawnMethodKey` with `'subint'` literal
  - Add `case 'subint':` to `try_set_start_method()`
    with `sys.version_info < (3, 14)` guard
  - Conditionally import `subint_proc` into `_methods`
    dict (version-gated)

- `tractor/spawn/_entry.py` - add `_subint_main()`:
  - Accepts simple args: `actor_name`, `actor_uuid`,
    `parent_addr_str`, `loglevel`, `infect_asyncio`
  - Constructs fresh `Actor` instance
  - Sets `_state._current_actor`
  - Calls `trio.run(partial(async_main, actor, ...))`
  - Same try/except/finally pattern as `_trio_main()`

- `tractor/runtime/_runtime.py`:
  - `_from_parent()` line 873: change
    `self._spawn_method == "trio"` to
    `self._spawn_method in ("trio", "subint")`
  - `load_modules()` line 556: same broadening

- `tractor/runtime/_supervise.py`:
  - Update `_children` type hint (line 115) to include
    `SubintProcess` (conditional import under
    `TYPE_CHECKING`)

## Step 6: Smoke testing + iteration

Run `pytest tests/test_spawning.py --spawn-backend subint`
on Python 3.14 and fix issues. Known areas to address:

- **Signal handling**: subint actors share the parent's
  signal handlers; skip custom SIGINT install for subints
- **`is_root_process()`**: the second def at
  `_state.py:232` uses `_runtime_vars['_is_root']` (not
  mp check), so subint children correctly report `False`
- **Debug mode**: skip `debug_mode=True` for subint
  backend initially (TTY lock protocol assumes separate
  procs)
- **IPC transport**: default subint actors to UDS
  transport (localhost-only, lower overhead)

## Codebase exploration summary

Key files explored during planning:

- `tractor/spawn/_spawn.py` - monolithic 848-line spawn
  module with `trio_proc()` (lines 448-670), `mp_proc()`
  (lines 672-839), shared helpers, and `_methods` dispatch
  dict
- `tractor/spawn/_entry.py` - subprocess entry points
  `_mp_main()` and `_trio_main()`
- `tractor/spawn/_forkserver_override.py` - stdlib
  forkserver patches
- `tractor/spawn/_mp_fixup_main.py` - `__main__` fixup
  with subint TODO comment
- `tractor/runtime/_runtime.py` - `Actor._from_parent()`
  spawn-method-gated `SpawnSpec` receive
- `tractor/runtime/_supervise.py` - `ActorNursery._children`
  type hint `trio.Process | mp.Process`
- `tractor/runtime/_state.py` - `is_root_process()` uses
  `_runtime_vars['_is_root']` (safe for subints)
- `tractor/_testing/pytest.py` - `--spawn-backend` CLI
  option, fixtures, `pytest_configure()`
- `tractor/ipc/_uds.py` - existing UDS transport (ready
  for subint IPC)
- `.github/workflows/ci.yml` - commented `subint` entries
  in test matrix
- Python 3.14 `concurrent.interpreters` API: `create()`,
  `exec()`, `call_in_thread()`, `close()`,
  `prepare_main()`, `create_queue()` - no native async
  support
