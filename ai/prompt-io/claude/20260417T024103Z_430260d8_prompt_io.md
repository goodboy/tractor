## Prompt

User pointed to the design prompt at
`ai/prompt-io/prompts/subints_spawner.md` which
references GH issue #379. The prompt requests:

1. Modularize the `tractor/spawn/_spawn.py` monolith
   by extracting backends into separate submodules
2. Add a new `_subint.py` backend using Python 3.14+
   `concurrent.interpreters` for subinterpreter-based
   actor spawning
3. Wire the test harness to parameterize all tests
   with the new `subint` backend via
   `--spawn-backend subint`

This is a planning-only session; the user intends to
upgrade the model and compare plans before
implementing.

## Response summary

Produced a 6-step implementation plan covering:

- **Steps 1-2**: Extract `trio_proc()` to `_trio.py`
  and `mp_proc()` to `_mp.py`, leaving shared helpers
  (`soft_kill`, `hard_kill`, `cancel_on_completion`,
  `exhaust_portal`, `new_proc` dispatcher) in
  `_spawn.py`
- **Step 3**: New `_subint.py` with `SubintProcess`
  wrapper class (process-like interface over
  interpreter+thread) and `subint_proc()` async
  function using `interp.exec()` bootstrap strategy
- **Step 4**: Wire `SpawnMethodKey`, `_methods` dict,
  `try_set_start_method()`, add `_subint_main()` entry
  point, broaden spawn-method checks in
  `_runtime.py`
- **Step 5**: Test harness `--spawn-backend subint`
  support
- **Step 6**: Smoke testing with known gotchas
  (signal sharing, debug mode, UDS default)

Key design decision: use `interp.exec(code_str)` with
a `threading.Thread` rather than
`interp.call_in_thread()` for more reliable
cross-interpreter bootstrap.
